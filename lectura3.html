BASES DE DATOS

El témino de bases de datos fue escuchado por primera vez en el año 1963, en la ciudad de California USA. Las bases de datos son un conjunto de datos relacionados agrupados o estructurados, que permiten al usuario guardar una gran cantidad de información de una manera organizada para facilitar su utilización. Estan compuestas de una o una serie de tablas que guardan una parte de la información sobre cada elemento que el susuario desee almacenar en la tabla, se debe tener en cuenta que cada fila de la tabla conforma un registro de la misma.

1. CARACTERÍSTICAS

Dentro de las características más relevantes en el manejo de bases de datos, podemos encontrar:

-Independencia lógica y física de los datos.

-Redundancia mínima.

-Acceso concurrente por parte de múltiples usuarios.

-Integridad de los datos.

-Consultas complejas optimizadas.

-Seguridad de acceso y auditoría.

-Respaldo y recuperación.

-Acceso a través de lenguajes de programación estándar.

2. VENTAJAS

Dentro de las ventajas en le manejo de base de datos tenemos:

-Control sobre la redundancia de datos: Con relación a los ficheros de datos los cuales almacenan varias copias de los mismos datos en ficheros distintos; las bases de datos no desperdician espacio de almacenamiento ya que se encuentran integrados.

-Consistencia de datos:Mediante el control de redundancias de datos, se reduce en gran medida el riesgo de que existan inconsistencias.Teniendo en cuenta que si un dato se almacena una sola vez, las actualizaciones se realizan una sola vez y las actualizaciones estan disponibles para todos los usuarios. Mientras que si un dato se encuentra duplicado y el sistema conoce esta redundancia, el mismo sistema debe encargarse de garantizar que todas las copias se mantienen consistentes.

-Compartir datos: En comparación con el sistema de ficherios donde los ficheros pertenecen a las personas o departamentos que los utilizan, las bases de datos pertenecen a la empresa y pueden ser compartida por todos los usuarios que se encuentren autorizados.

-Mantenimiento de estándares: Gracias a la integración es más facil respetar los estándares necesarios, tanto los establecidos a nivel de la empresa como los nacionales e internacionales. Los estándares pueden establecerse sobre el formato de los datos con el fin de facilitar su intercambio, podemos encontrar estándares de documentos, procedimientos de actualización y reglas de acceso.

-Mejora en Integridad de datos: La seguridad de una base de datos es la protección de la información frente a usuatriso no autorizados para su manejo.

-Mejora de accesibilidad: Muchos sistemas de gestión de bases de datos proporcionan lenguajes de consulta de informes que le permiten al usuario realizar cualquier tipo de consulta sobre la base de datos.

-Mejora en productividad: Proporciona todas las rutinas de manejo de ficheros típicas de los programas de aplicación. Al disponer de estas funciones permite al programador centrarse en la función específica del usuario.

-Mejora en el mantenimiento: Los sistemas de gestión de bases de datos, separan las descripciones de los datos de las aplicaciones, este proceso se conoce como dependencia de datos.

-Aumento de concurrencia: Gestionan el acceso concurrente a la base de datos y garantiza que no ocurran problemas de este tipo.

3. TIPOS DE CAMPOS

Dentro de los tipos de campos más utilizados en la elaboración de bases de datos podemos encontrar:

-Numérico: entre los diferentes tipos de campos numéricos podemos encontrar enteros “sin decimales” y reales “decimales”.

-Booleanos: poseen dos estados: Verdadero “Si” y Falso “No”.

-Memos: son campos alfanuméricos de longitud ilimitada. Presentan el inconveniente de no poder ser indexados.

-Fechas: almacenan fechas facilitando posteriormente su explotación. Almacenar fechas de esta forma posibilita ordenar los registros por fechas o calcular los días entre una fecha y otra.

-Alfanuméricos: contienen cifras y letras. Presentan una longitud limitada (255 caracteres).

-Autoincrementables: son campos numéricos enteros que incrementan en una unidad su valor para cada registro incorporado.

4. TIPOS DE BASES DE DATOS:

-MySql: es una base de datos con licencia GPL basada en un servidor. Se caracteriza por su rapidez. No es recomendable usar para grandes volúmenes de datos.

-PostgreSql y Oracle: Son sistemas de base de datos poderosos. Administra muy bien grandes cantidades de datos, y suelen ser utilizadas en intranets y sistemas de gran calibre.

-Access: Es una base de datos desarrollada por Microsoft. Esta base de datos, debe ser creada bajo el programa access, el cual crea un archivo .mdb con la estructura ya explicada.

-Microsoft SQL Server: es una base de datos más potente que access desarrollada por Microsoft. Se utiliza para manejar grandes volúmenes de informaciones.

5. ESTRUCTURA

Posee un orden que debe ser cumplido para acceder a la información de manera coherente. Cada base de datos contiene una o más tablas que cumplen la función de contener los campos.Una base de datos posee el siguiente orden jerárquico:

-Tablas

-Campos

-Registros

-Lenguaje SQL
